## 스프링 AOP

AOP(Aspect Oriented Programming)은 관점 지향 프로그래밍으로 어떤 로직을 핵심적인 관점, 부가적인 관점으로 나누어 보고, 그 관점들을 각각 모듈화 하는 것을 의미한다.

핵심적인 관점에서의 로직은 비즈니스의 로직이 되고 부가적인 관점에서의 로직은 예외 처리, 로깅 등이 된다. 

### AOP 주요 개념

- Advice
    - 흩어진 관심사를 모듈화 하는 것, 부가 기능 그 자체
- Aspect
    - 부가 기능과 부가 기능을 어디에 지정할지 정의하는 것(Advice + Pointcut)
- Jointpoint
    - Adivce가 적용될 수 있는 모든 위치
    - 스프링 AOP는 프록시 방식을 사용하므로 조인 포인트는 항상 메서드 실행 시점으로 제한
- Pointcut
    - Jointpoint 중에서 구체적으로 Adivce가 실행될 지점을 정하는 것
    - 프록시를 사용하는 스프링 AOP는 메서드 실행 시점만 포인트 컷으로 선별 가능
- Target
    - 어드바이스를 받는 객체, 포인트 컷으로 결정
